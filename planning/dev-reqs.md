# Requirements

## Axioms to Follow

1. This is not for work related projects. No code repos.
2. This should be visually memorable without being overbearing or garish.
3. If feedback is implemented, it will be private and not publicly exposed except maybe the blog collections.
4. Design must be bespoke and functional. No boring shapes or uninspired interactivity, but it must be accessible and intuitive.
5. Accessibility MUST be comprehensive and easy to use/find
